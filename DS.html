Types of Data Structures:

1. Arrays: A collection of elements, each identified by an index or a key.
2. Linked Lists: Elements are stored in nodes, and each node points to the next one in the sequence.
3. Stacks: Follows the Last In, First Out (LIFO) principle. The last element added is the first one to be removed.
4. Queues: Follows the First In, First Out (FIFO) principle. The first element added is the first one to be removed.
5. Trees: Hierarchical data structures with a root element and branching into subtrees.
6. Graphs: A collection of nodes connected by edges, representing relationships between elements.
7. Hash Tables: Data structure that maps keys to values, allowing for efficient retrieval.


Importance of Data Structures:

1. Efficiency: Choosing the right data structure can significantly impact the efficiency of algorithms and operations.
2. Organization: Provides a way to organize and manage data logically.
3. Search and Retrieval: Different data structures excel in various search and retrieval operations.
4. Memory Management: Efficient use of memory is crucial, and data structures play a key role in this.
5. Optimized Algorithms: Many algorithms depend on the use of specific data structures for optimal performance.

Linked Lists

Linked list Definition: A linked list is a sequence of items, with each item connected to the next by a link (a pointer).
A linked list can grow and shrink as necessary while the program is running.
An item and its link to the next item form a node.

Linked lists are a dynamic data structure, which can grow and be pruned, allocating, and deallocating memory while the program is running.

Therefore, they are used to represent data with such behaviour, examples:
1- Dynamic data structures such as stacks and queues .
2- Pperating systems use doubly linked lists to maintain references to active processes, threads, and other dynamic objects.

Basic functionality:

Create a list
Count - returns the number of elements.
Add(T) - appends given element T at the end.
AddFirst(T) - appends given element at the First.
AddLast(T) (the same as ADD(T)).
Remove(T) - removes a given element from the end.

Queue DTS

Queue Definition: Container of objects that are inserted and removed according to the first-in first-out (FIFO) principle.
A data structure in which elements are added to the rear and removed from the front; a "first in, first out" (FIFO).
Container which provides a method, enqueue, for putting objects at the rear of the container, and a method, dequeue, for taking objects out of the container at the front.

Queue Methods:

1. enqueue(o) insert object o at the rear of the queue.
Input: Object; Output: None
2. dequeue() removes and return the object at the front of the queue; an error occurs if the:
Input: None; Output: Object
3. size( ) return the number of objects in the queue
Input: None; Output: Integer
4. isEmpty( ) return a Boolean indicating if the queue is empty
Input: None; Output: Boolean
5. front( ) returns the front object in the queue, without removing it; an error occurs if the queue is empty
Input: None; Output: Object


Stack DTS

Stack Definition: Is a container which provides exactly one method, push, for putting objects into the container, and one method, pop, for taking objects out of the container.

Is a container of objects that are inserted and removed according to the last-in first-out (LIFO) principle.
Objects which are stored in a stack are kept in a pile.
The last item put into the stack is on the top.
When an item is pushed into a stack, it is placed at the top of the pile.
When an item is popped, it is always the top item which is removed.

The main features of a stack: 1. Access is allowed at one end (the top of the stack) only.
Data are added and removed at this end.
2. The action of adding a data element to a stack is called pushing.
3. The action of removing a data element from a stack is called popping.
4. Stacks handle data in a last-in-first-out. or LIFO,
fashion: the last item pushed onto the stack is always the First item popped off the stack.

Stack Operations:

Construction: Initializes an empty stack
Push operation: Modifies a stack by adding a value to top of stack
Pop operation: Modifies a stack by removing the top value of the stack
Top operation: Retrieves the value at the top of the stack
IsEmpty operation: Determines if stack contains any values To help with debugging, add early on:
Size Op Retrieves the number of element of the stack
Output: Displays all the elements stored in the stack.

Graphs

A graph is a data structure that consists of a finite set of nodes (vertices) and a collection of edges that connect pairs of nodes. Graphs are used to represent relationships between different entities, making them a powerful and versatile data structure.

Key Components: 1. Nodes (Vertices): Represent entities or points in the graph.
2. Edges: Connect pairs of nodes and define relationships between them.
Types of Graphs: 1. Directed Graph (DiGraph): Edges have a direction, indicating a one-way relationship between nodes.
2. Undirected Graph: Edges do not have a direction, representing a symmetric relationship between nodes.
3. Weighted Graph: Assigns weights or costs to edges, indicating the strength or distance between nodes.
4. Cyclic Graph: Contains at least one cycle (a loop that begins and ends at the same node).
5. Acyclic Graph: Does not contain any cycles.
Representations:
1. Adjacency Matrix: A 2D array where each cell (i, j) represents an edge between nodes i and j.
2. Adjacency List: A collection of lists or arrays, where each list represents the neighbors of a particular node.
Applications:
1. Networks: Representing connections in social networks, transportation systems, etc.
2. Routing Algorithms: Finding the shortest path between nodes.
3. Dependency Resolution: Managing dependencies in software projects.
4. Circuit Design: Modeling electrical circuits.
Graphs are fundamental in solving a wide range of problems and are extensively used in computer science, mathematics, and various real-world applications.
